  1. git reset --mixed chash
    可以更新commit记录并覆盖暂存区。

可以看到c1已经不被跟踪了

  1. git reset --hard chash
    上面这个指令可以使分支回滚到上一次提交记录上：

  2. git reset chash
    上面这个指令默认选项是mixed

总结一下：
• –soft – 缓存区和工作目录都不会被改变
• –mixed – 默认选项。缓存区和你指定的提交同步，但工作目录不受影响
• –hard – 缓存区和工作目录都同步到你指定的提交
